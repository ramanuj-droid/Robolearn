   - Two tabs (Login / Register)

3. home.html (User Dashboard)
   - Top section: User profile (name, email, maybe profile pic later)
   - Middle section: Roadmap Form
        - Inputs: skills, goals, pace, etc.
        - Button: "Generate Roadmap"
   - Bottom section (after generation): Display Roadmap
        - Step-by-step roadmap generated by AI
        - Options: Save, Regenerate, Export


ROADMAP GENERATION WORKFLOW

1 User Input (Form on home.html)

The user fills out a form with fields like:
Current skill level (e.g., beginner, intermediate).
Career goal (e.g., Frontend Developer, Data Scientist).
Preferred learning pace (hours per day/week).
Extra preferences (optional: resources type, preferred language, etc.).
Form → sends this data via POST /api/roadmap/generate to your backend.

---------------------------------------------------------------------------------------------------------------------------------------------------
Backend Receives Data

The backend takes the user input and prepares a prompt (structured text) for the AI. Example:
“Generate a learning roadmap for a beginner who wants to become a Data Scientist.
They can study 2 hours per day. Provide a weekly breakdown with Topics, Time Required, YouTube links, Documentation, and Practice Exercises.”

--------------------------------------------------------------------------------------------------------------------------------------------------------
3AI Processing (LLM API Call)

Backend calls an LLM (like OpenAI, Anthropic, or a free model like HuggingFace Falcon / Llama 3).
AI responds with a step-by-step roadmap (structured as JSON or Markdown).
Example JSON:

[
  { "week": 1, "topic": "Python Basics", "time": "10 hrs", "youtube": "...", "docs": "...", "practice": "Hackerrank" },
  { "week": 2, "topic": "Data Analysis with Pandas", "time": "8 hrs", "youtube": "...", "docs": "...", "practice": "Kaggle" }
]

--------------------------------------------------------------------------------------------------------------------------------------------------------

Backend Saves Roadmap

Roadmap is stored in MongoDB under the user’s profile.
Schema links roadmap to userId, so they can see it later.

--------------------------------------------------------------------------------------------------------------------------------------------------------

Frontend Displays Roadmap

Frontend renders the roadmap in a table format with:
Time | Topic | YouTube Link | Docs | Practice | Progress Checkbox

--------------------------------------------------------------------------------------------------------------------------------------------------------
Post-Generation Options

Progress Tracking: User checks off completed topics → backend updates DB.
Export: User clicks Export PDF/Excel → frontend generates file with clickable links.
Regenerate: User tweaks inputs → API regenerates roadmap without deleting old ones.
